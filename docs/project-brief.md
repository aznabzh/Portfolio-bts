# Project Brief

## Project Context
This project is a portfolio website created for the BTS SIO SLAM E5 exam.

The site is used as a presentation support during the oral exam in front of a jury.
The jury typically includes:
- two teachers
- one professional

This portfolio is exclusively dedicated to the BTS portfolio.

---

## Main Purpose
The goal of the site is to centralize everything achieved during the two years of BTS, in a clear and structured way.

The site must make it possible to present:
- projects completed during training
- projects completed during internship
- personal projects if relevant
- competencies worked on during each project
- proofs associated with those competencies
- technology watch

---

## Core Oral Presentation Need
During the oral exam, the important thing is not only to show projects.

The portfolio must make it easy to demonstrate:
- which competencies were worked on
- in which project
- with which concrete proofs

The site must therefore strongly support the relationship between:
- projects
- competencies
- proofs

This is one of the main functional goals of the project.

---

## Main User
Primary user:
- the student presenting the portfolio during the oral exam

Secondary users:
- members of the jury consulting the portfolio during the presentation

---

## Expected Experience
The site must help the student:
- navigate quickly during the oral
- find a project easily
- find a competency easily
- show proofs quickly
- move from matrix view to project details smoothly

The site should feel useful as a presentation tool, not just as a visual showcase.

---

## Content to Present
The site should include:
- a home page introducing the portfolio
- a projects page grouped by category
- a project detail page
- a competencies page with a competency matrix
- a technology watch page
- a simple contact page

---

## Project Categories
Projects may belong to categories such as:
- Ateliers professionnels / réalisations en formation
- Stages / réalisations en milieu professionnel
- Réalisations personnelles

---

## Style Direction
The site should feel:
- modern
- sober
- clean
- clear
- slightly developer-oriented
- elegant but not flashy
- simple but not simplistic

The desired tone is:
- serious
- readable
- structured
- visually polished

The site must avoid:
- startup landing page style
- overly corporate style
- editorial/luxury style
- flashy colors
- excessive animation
- empty layout with too much wasted space

---

## Visual Reference
The validated v0 mockup is the current visual reference.

The implementation should preserve:
- the overall aesthetic direction
- the page structure
- the layout rhythm
- the core visual language

The goal is not to redesign the site from scratch.
The goal is to build a cleaner and more maintainable implementation under the same visual shell.

---

## Technical Direction
The site should remain:
- simple
- mostly static
- maintainable
- data-driven

It should not include:
- backend
- database
- authentication
- admin dashboard
- CMS logic
- unnecessary complexity

---

## Success Criteria
The project is successful if:
- it remains visually close to the validated mockup
- it is easy to navigate during the oral exam
- it clearly shows projects, competencies, and proofs
- it is easy to maintain
- new projects and proofs can be added easily
- the site feels serious, coherent, and presentation-ready
